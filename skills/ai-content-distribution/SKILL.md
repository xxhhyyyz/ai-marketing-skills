# AI Content Distribution — AI 产品内容传播策略

AI 产品在不同渠道的传播方法论和执行 SOP。

**前置依赖**：先读取 [ai-marketing-context](../ai-marketing-context/SKILL.md) 了解产品背景。

**相关 Skills**：
- [ai-content-capture](../ai-content-capture/SKILL.md) — 传播前的素材准备
- [ai-event-sop](../ai-event-sop/SKILL.md) — 活动也是重要传播渠道
- [ai-hackathon](../ai-hackathon/SKILL.md) — 赛事传播策略

## 工作流程

```
明确传播目标 → 选择渠道组合 → 适配内容形式 → 规划发布节奏 → 执行 & 复盘
```

---

## Step 1：明确传播目标

根据用户意图，确定本次传播的核心目标：

| 目标类型 | 核心指标 | 适合渠道 |
|---------|---------|---------|
| 品牌认知 | 曝光量、媒体报道数 | 媒体、社交平台、行业活动 |
| 开发者获取 | 注册数、API 调用量 | GitHub、技术社区、Dev 博客 |
| 用户增长 | 注册转化率、激活率 | Product Hunt、社交广告、SEO |
| 生态建设 | 合作伙伴数、集成数 | 开发者关系、技术论坛 |
| 融资传播 | 媒体覆盖、行业讨论 | 科技媒体、Twitter/X、LinkedIn |

### 2.2 同一素材的多平台适配

一条核心素材应该被改写为不同形式：

```
核心素材：「新模型推理速度提升 40%」

→ Twitter/X：
  "Our new model is 40% faster. Here's what that means for your app. 🧵"
  [配 Demo GIF]

→ 微信公众号：
  「当推理速度提升 40%，你的 AI 应用能多做些什么？」
  [2000 字深度解读 + 技术原理 + 用户案例]

→ GitHub Release：
  "### Performance: 40% inference speed improvement
   - Benchmark details: ...
   - Migration guide: ..."

→ 知乎：
  回答「如何评价 XX 模型最新更新？」

→ Hacker News：
  "Show HN: [Product] – Now 40% faster inference with [technique]"

→ 小红书：
  「AI 又进化了！速度快了 40% 是什么概念？实测对比」
  [配对比截图]
```

## Step 3：AI 产品发布节奏

### 3.1 重大发布（新产品 / 大版本 / 融资）

```
T-14  预热期
├── 内部对齐：确认 messaging、FAQ、demo
├── 种子用户提前体验 + 收集反馈
└── 预告内容：悬念海报 / teaser 视频

T-7   蓄势期
├── KOL/媒体提前沟通 + 独家体验
├── 社区预告：「下周有大事发生」
└── 准备所有渠道的内容版本

D-Day 发布日
├── 08:00  官方博客/公众号发布
├── 09:00  Twitter/X thread + GitHub release
├── 10:00  Hacker News / Reddit / Product Hunt
├── 11:00  KOL 同步发布
├── 14:00  媒体报道跟进
└── 全天   社区互动、回复评论

T+3   扩散期
├── 用户 UGC 转发和引用
├── 二次解读文章（技术深度 / 行业分析）
├── 媒体采访和播客
└── 社区 AMA / Live Demo

T+14  长尾期
├── 用户案例和 success story
├── 教程和最佳实践
├── SEO 优化：长尾关键词文章
└── 数据复盘和策略调整
```

### 3.2 常规更新（小版本 / Feature 更新）

```
D-Day：
├── GitHub Release + Changelog
├── Twitter/X 1-2 条推文
├── 社区公告（Discord/Slack）
└── 公众号技术文章（可选）

T+3：
├── 使用教程
└── 回复社区反馈
```

### 3.3 日常内容（非发布期）

保持持续的内容输出，维持品牌存在感：

- **技术内容**：工程博客、架构解读、开源贡献
- **用户故事**：案例分享、用户访谈
- **行业洞察**：趋势分析、竞品评论（注意品牌策略）
- **社区互动**：回答问题、参与讨论
- **团队文化**：招聘、团队故事（适量）

## Step 4：开发者关系与 KOL 策略

### 4.1 AI 领域 KOL 类型

| 类型 | 特点 | 合作方式 |
|------|------|---------|
| **技术大 V** | 深度技术内容、高可信度 | 独家体验 + 技术评测 |
| **AI 创业者** | 真实使用场景 | 案例合作 + 联合内容 |
| **开发者博主** | 教程和实践内容 | 提供 API 额度 + 技术支持 |
| **科技媒体** | 行业报道、覆盖面广 | 新闻稿 + 独家采访 |
| **YouTube/B站 up 主** | 视频教程、影响力大 | 赞助视频 + 联合创作 |

### 4.2 KOL 合作流程

```
筛选匹配 → 首次触达 → 提供体验 → 内容共创 → 发布协调 → 效果追踪
```

**触达模板**：

```
Hi [Name],

我是 [公司] 的 [角色]。关注你在 [领域] 的内容很久了，
特别是你最近关于 [具体内容] 的分享。

我们最近发布了 [产品/功能]，[一句话说价值]。
想给你提供提前体验的机会，看看是否有合作空间。

[具体权益：API 额度 / 独家功能 / 技术支持]

方便的话聊一下？

Best,
[Name]
```

## Step 5：AI 搜索优化（AEO）

让 AI 搜索引擎（ChatGPT、Perplexity、Google AI Overview）引用你的产品：

### 5.1 核心策略

- **结构化内容**：FAQ 格式、对比表格、步骤指南
- **权威来源建设**：技术博客、论文引用、Benchmark 数据
- **语义覆盖**：覆盖用户可能问 AI 的问题
- **Schema 标记**：确保搜索引擎能正确解析

### 5.2 内容优化清单

```
- [ ] 产品落地页包含 FAQ 部分（覆盖常见问题）
- [ ] 技术博客有清晰的 H1-H3 层级结构
- [ ] 对比页面使用结构化表格（我们 vs 竞品）
- [ ] API 文档有完整的使用示例
- [ ] GitHub README 信息完整且最新
- [ ] 官网有独立的 "What is [产品]" 解释页面
- [ ] 内容中自然包含 "[产品] vs [竞品]" 类表述
- [ ] 用户案例包含具体数据和可验证的成果
```

## Notes

- AI 产品传播的核心是**让技术价值可感知**，避免堆砌技术参数
- 不同阶段传播重点不同：冷启动重社区，增长期重规模，成熟期重品牌
- 开发者是 AI 产品最重要的传播节点，他们既是用户也是推广者
- 国内外渠道策略差异大，注意区分中英文内容
- 所有对外内容需符合 ai-marketing-context 中的品牌调性要求
