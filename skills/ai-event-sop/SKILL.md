# AI Event SOP — AI 活动全流程标准化作业

AI 相关活动（峰会、开发者日、Workshop、Meetup、线上直播）的标准化筹备和执行流程。

## 工作流程总览

```
活动定位 → 立项审批 → 筹备执行 → 当天运营 → 复盘传播
```

## Step 1：活动类型决策

### 1.1 活动类型矩阵

| 类型 | 规模 | 时长 | 目标 | 预算 |
|------|------|------|------|------|
| **AI 峰会/大会** | 200-2000 人 | 1-2 天 | 品牌影响力 + 生态 | 高 |
| **开发者日** | 50-300 人 | 半天-1 天 | 开发者获取 + 技术布道 | 中高 |
| **技术 Workshop** | 20-50 人 | 2-4 小时 | 深度教学 + 用户激活 | 低 |
| **Meetup/沙龙** | 30-100 人 | 2-3 小时 | 社区建设 + 交流 | 低 |
| **线上直播/Webinar** | 不限 | 1-2 小时 | 规模传播 + 线索收集 | 极低 |
| **闭门会** | 10-30 人 | 2-4 小时 | 高端客户 + 深度交流 | 中 |

### 1.2 快速决策树

```
目标是什么？
├── 品牌认知 → AI 峰会 / 大会
├── 开发者获取 → 开发者日 / Workshop
├── 用户教育 → Workshop / Webinar
├── 社区维护 → Meetup / 沙龙
├── 线索收集 → Webinar / 闭门会
└── 生态合作 → 峰会 / 闭门会

预算？
├── 充足 → 线下大会
├── 中等 → 开发者日 / Meetup
├── 有限 → Workshop / Webinar
└── 零预算 → 线上直播 + 社区联办
```

## Step 3：全流程 Checklist

### T-30（活动前 30 天）

```markdown
### 立项与规划
- [ ] 活动立项文档完成并审批
- [ ] 确定日期、时间、地点
- [ ] 预算审批通过
- [ ] 成立筹备小组，明确分工

### 场地与基建
- [ ] 场地预订确认（含备选方案）
- [ ] 场地实地踩点（容量、AV 设备、网络、电源）
- [ ] 网络带宽确认（AI Demo 需要稳定网络）
- [ ] 直播/录制方案确认

### 内容与嘉宾
- [ ] 议题框架设计
- [ ] 嘉宾候选名单（至少 2 倍于需求）
- [ ] 嘉宾邀请函发出
- [ ] 内部分享议题确认
```

### T-14（活动前 14 天）

```markdown
### 嘉宾确认
- [ ] 所有嘉宾确认参加
- [ ] 收集嘉宾 PPT/材料
- [ ] 嘉宾 bio 和照片收集
- [ ] 嘉宾差旅/住宿安排

### 物料设计
- [ ] 主视觉设计定稿
- [ ] 议程海报
- [ ] 易拉宝 / 背景板设计
- [ ] 胸牌 / 签到台设计
- [ ] 周边礼品准备（贴纸、T恤等）

### 推广启动
- [ ] 报名页面上线
- [ ] 推广文案定稿（多渠道版本）
- [ ] 社交媒体推广计划执行
- [ ] KOL / 社区渠道推广
- [ ] 邮件邀请发送
```

```markdown
- [ ] 主持人串词和 flow 定稿

### 技术测试

- [ ] 直播推流测试
- [ ] 录制设备测试
- [ ] 签到系统测试

### 运营准备
- [ ] 报名数据确认 + 到场率预估
- [ ] 餐饮下单（如有）
- [ ] 志愿者/工作人员培训
- [ ] 应急预案文档
```

### T-1（活动前 1 天）

```markdown
### 最终确认
- [ ] 场地布置完成
- [ ] AV 设备最终测试
- [ ] 所有 PPT 拷贝到现场设备
- [ ] Demo 最终测试（用现场网络）

- [ ] 工作人员到位确认
```

### D-Day（活动当天）

```markdown
### 开场前（提前 2 小时）
- [ ] 场地巡检
- [ ] AV 设备开机测试
- [ ] 网络连通性确认
- [ ] 签到台就位
- [ ] 摄影/录制团队就位
- [ ] 嘉宾到场接待

### 活动进行中
- [ ] 签到引导
- [ ] 主持人控场
- [ ] 技术支持随时待命（尤其 Demo 环节）
- [ ] 社交媒体实时图文直播
- [ ] 拍照记录（嘉宾、观众、互动、场景）
- [ ] 控制时间节奏

### Demo 环节特别注意
- [ ] 提前 15 分钟准备 Demo 环境
- [ ] 准备离线备份方案（录屏/截图）
- [ ] 安排技术人员在旁协助

### 收尾
- [ ] 感谢嘉宾和参会者
- [ ] 收集现场反馈（扫码问卷）
```

```markdown
- [ ] 精选照片整理发布

- [ ] 嘉宾演讲视频剪辑

- [ ] 参会者感谢邮件
- [ ] 新注册用户跟进
- [ ] 意向客户移交销售
```

## Step 4：AI Demo 环节设计

AI 产品活动中 Demo 是核心环节，需要特别设计：

### Demo 设计原则

1. **从用户场景出发**，而非技术能力展示
2. **实时互动** > 录播演示（但准备录播备份）
3. **观众参与**：让现场观众提供输入
4. **Before/After 对比**：先展示痛点，再展示解决方案
5. **控制时长**：单个 Demo 5-10 分钟，不超过 15 分钟

### Demo 准备清单

```markdown
- [ ] Demo 脚本编写（每一步操作 + 话术）
- [ ] 网络方案：主线 + 备用热点 + 离线方案
- [ ] API 额度确认（不要活动当天超限）
- [ ] 提前准备好示例输入和预期输出
- [ ] 录制一份完整的 Demo 视频作为 Plan B
- [ ] 准备「哇点」：一个出乎意料的效果
- [ ] 排练至少 3 次
```

## Step 6：一场活动产出 10+ 内容

**内容最大化策略**：

```
一场 2 小时活动 →

1. 活动预告文章
2. 倒计时海报（3 张）
3. 现场图文直播（Twitter/朋友圈）
4. 活动回顾长文
5. 每位嘉宾的演讲视频（独立剪辑）
6. 精华片段短视频（3-5 条，15-60 秒）
7. 嘉宾金句海报（每人 1 张）
8. PPT 资料合集
9. Q&A 精选整理
10. 活动数据复盘文章
11. 参会者证言收集
12. 下次活动预告
```

## Notes

- AI 产品活动的核心差异化是**现场 Demo**，务必投入足够准备
- 网络是 AI Demo 的生命线，永远准备备份方案
- 活动内容的二次传播价值往往大于活动本身
- 小而精的活动（Workshop、Meetup）比大而空的活动更有效
- 每次活动都是素材机会：照片、视频、证言、数据
