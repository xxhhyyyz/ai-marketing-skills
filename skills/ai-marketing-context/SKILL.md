## 工作流程

```
检查上下文文件是否存在
  → 存在：读取并确认是否需要更新
  → 不存在：引导用户创建
```

### Step 1：检查上下文文件

读取 `.agents/ai-marketing-context.md`（或回退路径）。

- 如果文件存在且内容完整，提示用户：
  ```
  ✅ 已找到 AI 产品营销上下文，上次更新于 <date>。
  需要更新吗？还是直接开始工作？
  ```
- 如果文件不存在或为空，进入 Step 2。

### Step 2：引导用户创建上下文

通过对话收集以下信息，逐步引导，不要一次性要求全部填写。

#### 2.1 产品基础信息

```markdown
## 产品基础
- **产品名称**：
- **一句话定位**：（20 字以内描述产品是什么）
- **产品形态**：API / SDK / SaaS / 开源项目 / 硬件 / 混合
- **核心能力**：（列出 3-5 个核心技术能力）
- **商业模式**：免费 / Freemium / 订阅制 / 按量计费 / 企业定制
```

#### 2.2 AI 行业特有字段

```markdown
## AI 特有属性
- **模型类型**：LLM / 多模态 / 语音 / 视觉 / Agent / 其他
- **模型来源**：自研 / 基于开源微调 / 调用第三方 API
- **开源策略**：完全开源 / 部分开源 / 闭源
- **技术壁垒**：（什么能力是竞品短期内难以复制的）
- **Benchmark 表现**：（如有，列出关键评测成绩）
```

#### 2.3 目标用户

```markdown
## 目标用户
- **主要用户画像**：
  - 角色：开发者 / 产品经理 / 企业决策者 / 创作者 / 其他
  - 技术水平：初学者 / 中级 / 高级
  - 行业：通用 / 金融 / 医疗 / 教育 / 电商 / 其他
- **次要用户画像**：（如有）
- **用户痛点 Top 3**：
  1.
  2.
  3.
- **用户在哪里**：GitHub / Twitter / 微信 / 知乎 / HN / Reddit / Discord / 其他
```

#### 2.4 竞品格局

```markdown
## 竞品分析
| 竞品 | 定位 | 优势 | 劣势 | 我们的差异化 |
|------|------|------|------|-------------|
|      |      |      |      |             |

- **竞争策略**：正面竞争 / 差异化定位 / 细分市场 / 生态整合
```

#### 2.5 品牌与内容

```markdown
## 品牌调性
- **品牌人格**：（如：专业可信 / 极客酷炫 / 亲和温暖 / 硬核技术）
- **内容风格**：
  - 技术深度：深度技术 / 技术科普 / 非技术友好
  - 语气：正式 / 半正式 / 轻松 / 幽默
  - 视觉风格：简约 / 科技感 / 多彩活泼
- **禁用词/话题**：（品牌不希望关联的词汇或话题）
- **竞品提及策略**：主动对比 / 避免提及 / 仅在被问时回应
```

#### 2.6 当前营销重点

```markdown
## 当前阶段与重点
- **产品阶段**：冷启动 / 增长期 / 成熟期 / 品牌升级
- **本季度核心目标**：（如：开发者注册量、API 调用量、品牌认知度）
- **近期重点事件**：（新版本发布、融资、活动、合作）
- **营销预算级别**：零预算 / 小额 / 中等 / 充足
```

### Step 3：生成上下文文件

将收集到的信息整合为结构化 Markdown 文件，写入 `.agents/ai-marketing-context.md`。

文件顶部添加元信息：

```markdown
---
product: <产品名称>
updated: <YYYY-MM-DD>
stage: <产品阶段>
---
```

### Step 4：确认与迭代

生成后展示给用户确认：

```
📋 AI 产品营销上下文已生成！

请检查以下部分是否准确：
1. 产品定位和核心能力
2. 目标用户画像
3. 竞品差异化
4. 品牌调性

有需要修改的地方告诉我，或者确认后就可以开始使用其他 AI Marketing skill 了。
```

## 被其他 Skill 引用时的行为

当其他 AI Marketing skill（如 ai-content-capture）启动时：

1. 自动读取 `.agents/ai-marketing-context.md`
2. 如果文件不存在，提示：
   ```
   ⚠️ 还没有设置 AI 产品营销上下文。
   建议先运行 /ai-marketing-context 创建基础信息，
   这样后续内容会更贴合你的产品。
   
   现在创建？还是跳过，直接开始？
   ```
3. 从上下文中提取当前任务需要的关键信息（产品名、定位、目标用户、品牌调性）

---

## 更新上下文

用户可以随时说"更新营销上下文"或"更新产品定位"来触发更新流程：

1. 读取现有文件
2. 询问用户要更新哪些部分
3. 仅修改指定部分，保留其他内容
4. 更新 `updated` 日期

## 飞书 CLI 集成

如果安装了飞书 CLI（`lark-cli`），上下文创建完成后可以自动同步：

```bash
# 将上下文文档同步到飞书云文档，方便团队协作
lark-cli docs +create --markdown .agents/ai-marketing-context.md --title "AI 产品营销上下文"

# 在飞书群里通知团队上下文已更新
lark-cli im +messages-send --chat-id oc_xxx --text "📋 营销上下文已更新，请查阅最新版本"
```

上下文不应该只活在本地文件里——同步到飞书，让整个市场团队都对齐。

## 野路子

- **别做问卷调查式的信息收集**。如果用户嫌烦，直接说「你就当我是新来的实习生，用大白话给我讲讲咱们产品」，比填表格出的信息量大 10 倍
- **上下文里最重要的不是你写了什么，而是你的「禁用词」和「竞品提及策略」**。这两个字段决定了 Agent 不会帮你惹麻烦
- **每次团队开完市场策略会，顺手更新一下上下文**。会上达成的共识如果不落到这个文件里，Agent 下次还是按老的来
- **把竞品分析当情报而非文档来维护**。竞品有动作了就立刻更新，别等到「下次 review」

## Notes

- 上下文文件是所有 AI Marketing skill 的「单一事实来源」
- 保持上下文文件简洁，每个字段都应该对营销决策有直接帮助
- 建议每月或每次重大产品更新后 review 一次
- 上下文不替代详细的产品文档，只记录营销相关的核心信息
